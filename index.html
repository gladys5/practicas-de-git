Practicas Git

Para que git vea una carpeta en particular usamos git init

git monitorea linea por linea

git status para revisar si el repositorio tiene cambios

configuracion inicial

git config --local user.name "Tu nombre aquí"

git config --local user.email "Tu@email.aqui”

si quieres que todos los archivos sean manejados por el mismo creador cambia local por global

para verificar que los archivos fueron agregados correctamente se comprueva con

git config user.name

git config user.email

HEAD: Estado actual de nuestro código, es decir, donde nos colocó Git Working tree: Lugar donde los archivos realmente están siendo almacenados index: Lugar donde Git almacena lo que será commiteado, es decir, la ubicación entre el working tree y el repositorio de Git en sí.

el comando: git log nos regresa el historico de las modificasiones:

commit c5caeb43sdfdsfgfdghdghjgwe345f9ab6 (HEAD -> master) Author: gladys5 Date: Mon Dec 12 03:56:03 2022 -0700 primer comit

el hash identificador unico del commit con este podras regresar a un punto si te equivocaste

git log cheatsheet

git init --bare

creamos un repositorio que no tendrá la working tree, o sea, no contendrá una copia de nuestros archivos. Como el repositorio servirá solo como servidor, para que otros miembros del equipo sincronicen sus trabajos, disminuimos espacio de almacenamiento de esta forma.


Antes de sincronizar nuestros cambios en el código con algún repositorio remoto, necesitamos agregarlos a nuestro repositorio local.


git remote add nombre-repositorio camino/del/repositorio

De esta forma tendremos un link de nuestro repositorio local con el repositorio remoto, que llamamos de nombre-repositorio, que está almacenado en camino/del/repositorio.


git push [repositorio] master

De esta forma enviamos las modificaciones de nuestra Branch master (hablaremos más sobre branches dentro de poco) hacia el repositorio remoto. Basta con substituir [repositorio] por el nombre que dimos al repositorio al agregarlo. Ahora, para traer los datos que están en el repositorio remoto, podemos utilizar git pull [repositorio] master
